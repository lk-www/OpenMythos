# Attention の効率化手法まとめ

---

## 1. ヘッドとは

Self-Attention を**複数の視点から並列に実行する**仕組みの、それぞれ1つの単位。

### 直感的なたとえ

同じ文を複数の専門家が読む場面を想像する。

```
"The cat sat on the mat"

専門家1（構文担当）: "sat" は "cat" の動詞だ
専門家2（意味担当）: "mat" は場所を表す
専門家3（照応担当）: "the" は既出の何かを指す
専門家4（語順担当）: 主語→動詞→場所の順だ
```

それぞれ**異なる観点**で文を読み、最後に意見をまとめる。これがマルチヘッドの発想。

### 仕組み

```
入力 x（次元 d_model）
  ↓ 線形変換で小さなベクトルに分割
  ↓
├─ ヘッド1: Q1,K1,V1（次元 d_k）→ Attention → 出力1
├─ ヘッド2: Q2,K2,V2（次元 d_k）→ Attention → 出力2
│   ...
└─ ヘッドH: QH,KH,VH（次元 d_k）→ Attention → 出力H
                                        ↓
                                   concat（結合）
                                        ↓
                                   × W_O（線形変換）
                                        ↓
                                    最終出力（次元 d_model）
```

次元の関係：

```
d_k = d_model / H

例）d_model=512、H=8 なら d_k=64
    各ヘッドは64次元で計算 → 結合すると512次元に戻る
```

計算量は1ヘッドのときと**ほぼ同じ**に保たれる。

### 各ヘッドが学習すること

| ヘッドの種類（観察例） | 役割 |
|---|---|
| 構文ヘッド | 主語・動詞の関係を捉える |
| 位置ヘッド | 隣接トークンへの注目 |
| 照応ヘッド | 代名詞と指示対象の対応 |
| 語彙ヘッド | 意味的に近い単語への注目 |

## 2. Grouped Query Attention（GQA）

### 背景：MHA の問題

標準の Multi-Head Attention（MHA）はヘッド数分の K・V を持つ。

```
MHA（ヘッド数 = 8 の場合）

Q: [Q1, Q2, Q3, Q4, Q5, Q6, Q7, Q8]
K: [K1, K2, K3, K4, K5, K6, K7, K8]  ← Q と同数
V: [V1, V2, V3, V4, V5, V6, V7, V8]  ← Q と同数
```

KV Cache に保存する量もヘッド数分になるため、メモリが膨大になる。

### 3つの方式の比較

**MHA（Multi-Head Attention）**
```
Q1 → K1, V1
Q2 → K2, V2
Q3 → K3, V3
Q4 → K4, V4
```
各 Q が専用の K・V を持つ。表現力は高いがメモリが重い。

**MQA（Multi-Query Attention）**
```
Q1 ─┐
Q2 ─┼→ K, V（1組だけ）
Q3 ─┤
Q4 ─┘
```
全 Q が1組の K・V を共有。メモリは最小だが品質が落ちやすい。
GQA reduces KV cache by having fewer KV heads than Q heads (factor of n_heads / n_kv_heads). 

**GQA（Grouped Query Attention）**
```
Q1, Q2 ─→ K1, V1（グループ1）
Q3, Q4 ─→ K2, V2（グループ2）
```
Q をグループに分け、グループ内で K・V を共有。MHA と MQA の中間。

### メモリ削減量（ヘッド数 H=8、グループ数 G=2 の場合）

| 方式 | K・V のヘッド数 | MHA 比 |
|---|---|---|
| MHA | 8 | 100% |
| GQA (G=2) | 2 | 25% |
| MQA | 1 | 12.5% |

### なぜ GQA が選ばれるか

| | MHA | GQA | MQA |
|---|---|---|---|
| 品質 | 最高 | MHA に近い | 落ちやすい |
| メモリ | 重い | 軽い | 最軽量 |
| 速度 | 遅い | 速い | 最速 |

MQA は品質の劣化が大きく、MHA はメモリが重すぎる。GQA はその**スイートスポット**。

### 採用モデル

| モデル | 方式 |
|---|---|
| GPT-3 | MHA |
| LLaMA 2 (70B) | GQA |
| LLaMA 3 | GQA |
| Mistral 7B | GQA |
| Gemma | GQA |

現代の大規模モデルは GQA がデファクトスタンダードになっている。

## 3. Multi-Latent Attention（MLA）

DeepSeek V2 で提案。GQA より根本的なアプローチで、**K・V 自体を低次元の潜在ベクトル（latent）に圧縮**してキャッシュする。

### 核心アイデア

```
通常の KV Cache:
  K, V（高次元）をそのままキャッシュ → メモリ大

MLA:
  K, V を生成する前の「圧縮ベクトル c」をキャッシュ → メモリ小
  推論時に c から K, V を復元して使う
```

### 仕組み

**通常の Attention**
```
x → W_K → K
x → W_V → V
x → W_Q → Q

Q × K^T → weights → × V → 出力
```

**MLA**
```
x ──→ W_down ──→ c（低次元潜在ベクトル）──→ W_up_K ──→ K
                  ↑                      └──→ W_up_V ──→ V
              ここだけキャッシュ

x ──→ W_Q ──→ Q（通常通り）
```

エンコーダ・デコーダ構造に似た**ボトルネック構造**。

### 次元の比較

```
d_model = 5120（例：DeepSeek V2）
通常の KV:  d_model × 2（K+V）× layers = 大
MLA の c:   d_c（圧縮次元、例：512）× layers = 小

削減率 ≈ d_c / (d_model × 2) = 512 / 10240 ≈ 5%
```

### RoPE との組み合わせ問題と解決策

RoPE は K に位置情報を埋め込むが、MLA は K を圧縮ベクトルから復元するため位置情報を圧縮ベクトルに含められない。

**解決策：デカップルド RoPE**

```
K を2つに分離：
  K_content  ← c から復元（位置情報なし）→ キャッシュ対象
  K_rope     ← 位置情報のみ（毎回計算）  → キャッシュしない

Q も同様に分離して対応
```

### 採用モデル

| モデル | 方式 |
|---|---|
| DeepSeek V2 / V3 | MLA |
| DeepSeek R1 | MLA |
| LLaMA 3 | GQA |
| Mistral | GQA |

## 4. 圧縮ベクトル c の算出方法

### 一言で言うと

**線形変換（行列積）で次元を落とすだけ**。

```
c = x × W_down

x      : (d_model)       例）5120次元
W_down : (d_model × d_c) 例）5120 × 512
c      : (d_c)           例）512次元
```

### 具体的な流れ

```
x（5120次元）
│
│  × W_down（5120 × 512）
│
↓
c（512次元）  ← 情報を圧縮した潜在ベクトル
│
├─ × W_up_K（512 × d_k）→ K
└─ × W_up_V（512 × d_v）→ V
```

### W_down は学習で決まる

```
学習の目的：
  圧縮しても K・V を復元したとき
  Attention の出力が正しくなるように W_down と W_up を同時に学習
```

「どの方向に圧縮すれば情報損失が最小か」をモデル自身が学ぶ。

### PCA との違い

| | PCA | MLA の W_down |
|---|---|---|
| 目的 | 分散を最大化する方向に圧縮 | Attention 出力の損失を最小化するように圧縮 |
| 学習 | データ統計から計算 | End-to-end で学習 |
| 最適化基準 | 入力の再構成誤差 | タスクの損失関数 |

MLA の圧縮は**タスクに特化した圧縮**。

### キャッシュとの関係

```
【推論時の流れ】

トークン t が来るたびに：

1. c_t = x_t × W_down      ← 小さい（512次元）のでキャッシュ
2. K_t = c_t × W_up_K      ← 使うときだけ復元
3. V_t = c_t × W_up_V      ← 使うときだけ復元

キャッシュ: [c_1, c_2, c_3, ..., c_t]  ← 全部 512次元
```

### なぜ品質が保てるのか

```
d_c = 512 でも情報が保持される理由：

x（5120次元）の中には冗長な情報が多い
  └─ Attention に必要な情報は実際には低次元の部分空間に存在する
  └─ W_down がその部分空間を学習して抽出する
```

Transformer の表現は本質的に低ランク構造を持つことが多く、圧縮しても Attention に必要な情報はほぼ保持できる。

## 5. 全手法の進化まとめ

```
MHA（2017）
  └─ K・V をヘッド数分キャッシュ。品質最高、メモリ重い

MQA（2019）
  └─ K・V を1組に。軽いが品質低下

GQA（2023）
  └─ K・V のヘッド数をグループ数に削減。バランス型

MLA（2024）
  └─ K・V を圧縮ベクトルからキャッシュ。メモリ最小級、品質高い
```

| | キャッシュ対象 | メモリ | 品質 |
|---|---|---|---|
| MHA | K・V（全ヘッド） | 最大 | 最高 |
| MQA | K・V（1組） | 最小 | 低下しやすい |
| GQA | K・V（グループ数） | 中 | MHA に近い |
| MLA | 圧縮ベクトル c | 最小級 | MHA に匹敵 |

「どの情報をキャッシュするか」という問いへの、より根本的な回答が MLA。
